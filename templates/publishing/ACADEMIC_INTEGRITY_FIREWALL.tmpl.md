## 2) The Three Walls

### Wall 1: Data Wall

Controls what data, splits, and preprocessing artifacts may cross between projects.

| Category | Status | Conditions |
|----------|--------|-----------|
| **Raw datasets** (publicly available) | Transferable | Must cite source; verify licensing |
| **Data splits from prior project** | Transferable with provenance | MUST have SHA-256 verification; see PRIOR_WORK_REUSE §5 |
| **Preprocessing pipeline from prior project** | Transferable with disclosure | MUST document any changes; see DATA_CONTRACT §6 |
| **Private/proprietary datasets** | NOT transferable | May not share across project boundaries |
| **Labeled data created for another course** | Check policy | May violate self-plagiarism rules; verify with {{HONOR_CODE_REF}} |

**Verification:** `python scripts/verify_{{PRIOR_PROJECT}}_snapshot.py` exits 0 (if prior work reuse exists). All data sources cited in report.

### Wall 2: Code Wall

Controls what code may be reused, shared, or generated by AI tools.

| Category | Status | Conditions |
|----------|--------|-----------|
| **Your own prior project code** | Transferable with provenance | MUST use vendor snapshot pattern; MUST disclose in report |
| **Open-source libraries** (pip/conda) | Transferable | Pin versions; cite in environment spec |
| **Code from classmates** | NOT transferable | Violates honor code |
| **Code from online solutions** | NOT transferable | Violates honor code even with citation |
| **AI-generated code** | Transferable with disclosure | MUST comply with AI_DIVISION_OF_LABOR; MUST disclose in AI Use Statement |
| **Boilerplate/scaffolding** | Transferable | Standard patterns (argparse, logging) need no special disclosure |

**Verification:** Git history shows code provenance. `config/{{PRIOR_PROJECT}}_provenance.json` (if applicable) records all reused file hashes. AI Use Statement present in report.

### Wall 3: Content Wall

Controls what text, figures, and analysis may cross between projects or be AI-generated.

| Category | Status | Conditions |
|----------|--------|-----------|
| **Your own analysis prose** | Must be original | Self-plagiarism: prior project text may NOT be copied verbatim |
| **Figure/table designs** | Transferable as templates | Actual data and captions MUST be project-specific |
| **AI-generated prose** | Restricted | See AI_DIVISION_OF_LABOR; human MUST write all analysis/discussion |
| **AI-generated code comments** | Permitted | Factual docstrings and comments are acceptable |
| **Copied text from papers** | NOT transferable | Must paraphrase and cite; direct quotes require quotation marks |
| **Prior project report text** | NOT transferable | Must rewrite; cite prior work if referencing your own results |

**Verification:** Run plagiarism check on report text. Search for verbatim sentences from prior project report. AI Use Statement accurately reflects AI tool usage.

## 3) Transferable vs Prohibited List

### 3.1 Transferable Artifacts

These may be reused across projects with proper documentation:

| Artifact | Documentation Required |
|----------|----------------------|
| Raw datasets (public) | Citation in report + data source in REPRO |
| Data splits with provenance | PRIOR_WORK_REUSE §5 + provenance.json |
| Preprocessing code with provenance | Vendor snapshot + hash verification |
| Model architecture code (if required by project spec) | Vendor snapshot + disclosure in Methods |
| Governance templates and project structure | No disclosure needed (infrastructure, not content) |
| Environment configuration files | Version pinning + ENV_CONTRACT compliance |
| Utility functions (seeding, logging, metrics computation) | Vendor snapshot if from prior project |

### 3.2 Prohibited Transfers

These MUST NOT be reused:

| Artifact | Why Prohibited | Consequence |
|----------|---------------|-------------|
| Classmate's code | Honor code violation | Academic misconduct |
| Online solution code | Honor code violation | Academic misconduct |
| Prior project report text (verbatim) | Self-plagiarism | Grade penalty or misconduct |
| Experiment results from prior project (as new results) | Falsification | Academic misconduct |
| Another student's figures or analysis | Plagiarism | Academic misconduct |

## 4) Executable Verification Commands

Run these checks before submission to verify integrity compliance:

### 4.1 Data Provenance Check

```bash
# Verify all reused data has provenance
python scripts/verify_{{PRIOR_PROJECT}}_snapshot.py  # exits 0

# Verify raw data hashes match expected values
python scripts/check_data_ready.py  # exits 0
```

### 4.2 Code Provenance Check

```bash
# Verify vendor snapshot integrity (if prior work reuse exists)
python scripts/verify_{{PRIOR_PROJECT}}_snapshot.py  # exits 0

# List all files not authored in this project
git log --diff-filter=A --name-only --format="" | sort | head -20
# Review: are all external files documented in provenance records?
```

### 4.3 Content Originality Check

```bash
# Search for potential self-plagiarism (if prior project report exists)
# Compare against prior report for verbatim sentences
diff <(cat {{PRIOR_REPORT_PATH}} | tr '.' '\n' | sort) \
     <(cat {{CURRENT_REPORT_PATH}} | tr '.' '\n' | sort) | grep "^<" | head -20

# Verify AI Use Statement exists
grep -c "AI Use Statement" {{CURRENT_REPORT_PATH}}
```

### 4.4 AI Usage Verification

```bash
# Verify AI_DIVISION_OF_LABOR contract exists and is populated
cat docs/AI_DIVISION_OF_LABOR.md | head -5

# Verify AI Use Statement in report matches AI_DIVISION_OF_LABOR
grep -A 10 "AI Use Statement" {{CURRENT_REPORT_PATH}}
```

## 5) AI Tool Usage Boundaries

*(Cross-reference: see AI_DIVISION_OF_LABOR for full details.)*

### 5.1 Summary of Boundaries

| Task | AI Permitted? | Human Responsibility |
|------|:------------:|---------------------|
| Code scaffolding | Yes | Review and understand all generated code |
| Debugging assistance | Yes | Understand the root cause and fix |
| Data analysis (running scripts) | Yes | Interpret results and draw conclusions |
| Literature search | Yes | Verify sources exist and are correctly cited |
| Writing report prose | Restricted | Human MUST write all analysis, discussion, and conclusions |
| Generating figures | Yes (code) | Human reviews for correctness and interpretation |
| Editing/proofreading | Yes | Human verifies edits preserve intended meaning |

### 5.2 Disclosure Format

The AI Use Statement in the report MUST follow the format specified in AI_DIVISION_OF_LABOR. At minimum:

- Name each AI tool used
- Describe what tasks each tool performed
- Confirm that all analysis, interpretation, and conclusions are human-authored
